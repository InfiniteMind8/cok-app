# Multi-Factor Authentication (MFA) — City of Karis

## Overview

Two-factor authentication (2FA / MFA) is enforced for **MASTER_ADMIN** and **ADMIN** roles.
Residents and Visitors may enrol optionally.

MFA uses Time-based One-Time Passwords (TOTP) via any standard authenticator app
(Google Authenticator, Authy, 1Password, Microsoft Authenticator).

---

## Enforcement

| Role | MFA status |
|---|---|
| MASTER_ADMIN | **Required** — cannot access `/admin/*` without enrolment |
| ADMIN | **Required** — same enforcement |
| RESIDENT | Optional — CTA in `/profile` settings |
| VISITOR | Optional — CTA in `/profile` settings |
| VENDOR | Optional |

Enforcement is applied at the admin layout level (`app/(admin)/layout.tsx`).
Every admin route request calls `requireMfaEnrolled()` which queries Clerk's
Management API for `twoFactorEnabled` status. Unenrolled staff are redirected to
`/account/mfa-enroll`.

---

## Enrolment Flow

1. Navigate to `/account/mfa-enroll` (or be redirected automatically as staff).
2. Click **Get started** — Clerk generates a TOTP secret.
3. Scan the QR code in your authenticator app, or enter the manual key.
4. Enter the 6-digit code to verify the setup.
5. Save the 10 backup codes shown — each is single-use. Store them in a password
   manager or a printed copy in a safe location.
6. Confirm you have saved the codes and click **Done**.

---

## Backup Codes

- 10 codes are generated after successful TOTP verification.
- Each code is single-use. Once consumed, it is invalidated by Clerk.
- Backup codes cannot be recovered after the page is closed.
- A new set of codes can be generated by completing the enrolment flow again
  (after an admin resets your MFA — see below).

---

## If You Lose Your Authenticator

1. Use one of your saved backup codes to sign in via Clerk's sign-in flow.
2. Contact a **Master Admin** to reset your MFA. The admin will:
   - Navigate to **Data Directory → [Your Account] → Reset MFA** (D.4 feature).
   - Confirm the action — this writes an audit log entry.
   - You will receive an email notifying you of the reset.
3. After reset, re-enrol at `/account/mfa-enroll`.

---

## Admin MFA Reset (MASTER_ADMIN only)

MFA reset is intentionally admin-initiated only. Users cannot self-reset MFA.

**Mechanism (built in D.4):**
```
clerkClient().users.disableUserMFA(clerkId)
```

The reset action must:
- Require `MASTER_ADMIN` role.
- Write an audit log entry: `mfa.admin_reset` with actor, target user, and timestamp.
- Send the target user a reset notification email (via Resend).

---

## Clerk Dashboard Setup Required

For full MFA enforcement, ensure the following in the **Clerk Dashboard**:

1. **Two-factor authentication** must be enabled for the application instance
   (Authentication → Multi-factor → Enable).
2. **TOTP** and **Backup codes** strategies must be enabled.
3. Optional: configure **Require 2FA** for specific roles if using Clerk's built-in
   enforcement (supplements the application-layer middleware for defence in depth).

Without Clerk dashboard setup, the application-layer check (`requireMfaEnrolled`)
still enforces MFA but users will not be challenged at the Clerk sign-in page itself.
